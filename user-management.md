# User Management

<span class="utopia-user-management-scope"></span>

Use User Management guidance to maintain user accounts, keep employee information accurate, manage account status, and support role-based access across the system.

## Overview

<div class="utopia-user-management-grid">
  <section>
    <h3>Maintain Accounts</h3>
    <p>Create, update, review, deactivate, and maintain user records through administrative controls.</p>
  </section>

  <section>
    <h3>Keep Details Accurate</h3>
    <p>Ensure personal, employment, contact, team, and banking information stays current.</p>
  </section>

  <section>
    <h3>Support Access Control</h3>
    <p>Coordinate user status and role information with the correct system permissions.</p>
  </section>
</div>

## User Information

<div class="utopia-user-management-table">
  <div class="utopia-user-management-head">Information Type</div>
  <div class="utopia-user-management-head">Details</div>

  <strong>Personal Information</strong>
  <p>Name, email address, date of birth, gender, nationality, religion, and other profile details.</p>

  <strong>Team Information</strong>
  <p>Company, business unit, department, position, reporting line, and related team structure.</p>

  <strong>Employment Details</strong>
  <p>Employment type, working shift, start date, employment status, and related job information.</p>

  <strong>Contact and Emergency</strong>
  <p>Phone number, work email, personal email, emergency contact, and emergency relationship details.</p>

  <strong>Bank Information</strong>
  <p>Bank holder name, bank name, account number, and payment-related employee records.</p>
</div>

## Managing Users

<div class="utopia-user-management-table">
  <div class="utopia-user-management-head">Action</div>
  <div class="utopia-user-management-head">Purpose</div>

  <strong>Add User</strong>
  <p>Create a new user account and make sure the required user details are recorded correctly.</p>

  <strong>Update Profile</strong>
  <p>Edit user information when employee details, contact data, or team assignments change.</p>

  <strong>Manage Status</strong>
  <p>Activate, deactivate, or update users based on employment and access requirements.</p>

  <strong>Review Records</strong>
  <p>Check that user records remain consistent, complete, and aligned with internal data.</p>
</div>

## Access and Roles

<div class="utopia-user-management-table">
  <div class="utopia-user-management-head">Area</div>
  <div class="utopia-user-management-head">Use</div>

  <strong>Role Assignment</strong>
  <p>Assign roles based on job responsibilities so each user can access the correct workspace.</p>

  <strong>Permission Review</strong>
  <p>Review access regularly to ensure users only see the features and data they need.</p>

  <strong>Inactive Users</strong>
  <p>Deactivate users who are no longer active to keep the system secure and organised.</p>

  <strong>Role Changes</strong>
  <p>Update access when a user changes department, position, company, or responsibility.</p>
</div>

## Common Workflow

<div class="utopia-user-management-flow">
  <div>
    <strong>1</strong>
    <span>Confirm the user request or profile change that needs to be handled.</span>
  </div>
  <div>
    <strong>2</strong>
    <span>Review the user profile, employment details, team information, and current status.</span>
  </div>
  <div>
    <strong>3</strong>
    <span>Update the required information and confirm the role or access level is still correct.</span>
  </div>
  <div>
    <strong>4</strong>
    <span>Save the changes and keep the record aligned with HR, admin, and permission requirements.</span>
  </div>
</div>

## Key Responsibilities

<div class="utopia-user-management-table">
  <div class="utopia-user-management-head">Responsibility</div>
  <div class="utopia-user-management-head">Focus</div>

  <strong>Account Accuracy</strong>
  <p>Ensure every user is valid, properly registered, and linked to accurate profile information.</p>

  <strong>Record Maintenance</strong>
  <p>Keep user information updated whenever employment, contact, team, or banking details change.</p>

  <strong>Status Control</strong>
  <p>Deactivate inactive users and keep account status aligned with employment status.</p>

  <strong>Access Consistency</strong>
  <p>Coordinate user records with roles and permissions so access remains appropriate.</p>
</div>
