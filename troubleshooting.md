# Troubleshooting

<span class="utopia-troubleshooting-scope"></span>

Use this guide to resolve common UtopiaSpace issues before escalating to support. If a problem continues, prepare screenshots, user details, page name, and the steps that caused the issue.

  <a href="/faq">FAQ</a>
  <a href="/login-guide">Login Guide</a>
  <a href="/settings">System Settings</a>
</div>

## Before You Escalate

<div class="utopia-troubleshooting-grid">
  <section>
    <h3>Refresh and Retry</h3>
    <p>Reload the page, confirm your internet connection, and try the action again.</p>
  </section>

  <section>
    <h3>Check Access</h3>
    <p>Make sure your role, filters, status, and workspace access match the page you are using.</p>
  </section>

  <section>
    <h3>Capture Details</h3>
    <p>Take screenshots and note the page, user account, request ID, date, and steps taken.</p>
  </section>
</div>

## Login Issues

<div class="utopia-troubleshooting-list">
  <section>
    <h3>Unable to log in using Google account</h3>
    <div class="utopia-troubleshooting-detail">
      <strong>Possible causes</strong>
      <p>Incorrect Google account, account not registered, browser session issue, or missing access setup.</p>
    </div>
    <div class="utopia-troubleshooting-detail">
      <strong>Try this</strong>
      <p>Use your registered company Google account, sign out and sign in again, clear browser cache and cookies, then contact the administrator if access is still denied.</p>
    </div>
  </section>
</div>

## Page Not Loading

<div class="utopia-troubleshooting-list">
  <section>
    <h3>System page does not load or appears blank</h3>
    <div class="utopia-troubleshooting-detail">
      <strong>Possible causes</strong>
      <p>Slow internet, browser compatibility issue, temporary system error, expired session, or cached page state.</p>
    </div>
    <div class="utopia-troubleshooting-detail">
      <strong>Try this</strong>
      <p>Refresh the page, check your connection, try Chrome, sign in again, or wait and retry if the system is temporarily unavailable.</p>
    </div>
  </section>
</div>

## Missing Data or Modules

<div class="utopia-troubleshooting-list">
  <section>
    <h3>Unable to see certain modules or data</h3>
    <div class="utopia-troubleshooting-detail">
      <strong>Possible causes</strong>
      <p>Role restrictions, missing permissions, incorrect workspace, company filter, location filter, status filter, or date range.</p>
    </div>
    <div class="utopia-troubleshooting-detail">
      <strong>Try this</strong>
      <p>Check your assigned role, adjust filters, confirm you are in the correct space, then contact the administrator if access is required.</p>
    </div>
  </section>
</div>

## Taskboard Issues

<div class="utopia-troubleshooting-list">
  <section>
    <h3>Task is not appearing on Taskboard</h3>
    <div class="utopia-troubleshooting-detail">
      <strong>Possible causes</strong>
      <p>The task is assigned to another user, hidden by status filter, outside the selected date range, or under a different task status.</p>
    </div>
    <div class="utopia-troubleshooting-detail">
      <strong>Try this</strong>
      <p>Check To-do, Done, and KIV statuses, clear filters, search by task name, and verify the assigned user.</p>
    </div>
  </section>
</div>

## Calendar Issues

<div class="utopia-troubleshooting-list">
  <section>
    <h3>Events, leave, or birthdays are not visible</h3>
    <div class="utopia-troubleshooting-detail">
      <strong>Possible causes</strong>
      <p>Filters are enabled, company or location selection is incorrect, date is wrong, or data has not refreshed.</p>
    </div>
    <div class="utopia-troubleshooting-detail">
      <strong>Try this</strong>
      <p>Check company, location, leave, birthday, and event toggles. Select the correct date and refresh the page.</p>
    </div>
  </section>
</div>

## Meeting Notes Issues

<div class="utopia-troubleshooting-list">
  <section>
    <h3>Unable to save meeting notes</h3>
    <div class="utopia-troubleshooting-detail">
      <strong>Possible causes</strong>
      <p>Required fields are missing, linked task data is incomplete, session expired, or validation failed.</p>
    </div>
    <div class="utopia-troubleshooting-detail">
      <strong>Try this</strong>
      <p>Complete all required fields, check task assignees and due dates, save again, then refresh if the issue persists.</p>
    </div>
  </section>
</div>

## Payment and Approval Issues

<div class="utopia-troubleshooting-list">
  <section>
    <h3>Unable to approve or process requests</h3>
    <div class="utopia-troubleshooting-detail">
      <strong>Possible causes</strong>
      <p>Insufficient permissions, request is not in the correct stage, missing supporting details, or the request has already been processed.</p>
    </div>
    <div class="utopia-troubleshooting-detail">
      <strong>Try this</strong>
      <p>Confirm your role allows approval, check request status, review supporting details, and contact the administrator if the access looks incorrect.</p>
    </div>
  </section>
</div>

## Performance Issues

<div class="utopia-troubleshooting-list">
  <section>
    <h3>System is slow or unresponsive</h3>
    <div class="utopia-troubleshooting-detail">
      <strong>Possible causes</strong>
      <p>High system usage, unstable network, too many browser tabs, large filters, or temporary backend delay.</p>
    </div>
    <div class="utopia-troubleshooting-detail">
      <strong>Try this</strong>
      <p>Refresh, close unused tabs, reduce filters, check your connection, and try again later if usage is high.</p>
    </div>
  </section>
</div>

## Wiki Sync Issues

<div class="utopia-troubleshooting-list">
  <section>
    <h3>Wiki page does not update after Git changes</h3>
    <div class="utopia-troubleshooting-detail">
      <strong>Possible causes</strong>
      <p>Wiki.js Git sync has not run yet, local Wiki.js edits created a merge conflict, or the local repository is stuck in a rebase.</p>
    </div>
    <div class="utopia-troubleshooting-detail">
      <strong>Try this</strong>
      <p>Use Import Everything or Force Sync. If Git reports a conflict, purge the local repository and import from GitHub as the source of truth.</p>
    </div>
  </section>
</div>

## When to Contact Support

<div class="utopia-troubleshooting-list">
  <section>
    <h3>Escalate if the issue continues after the basic checks</h3>
    <div class="utopia-troubleshooting-detail">
      <strong>Include these details</strong>
      <p>Page name, user account, role, company, date and time, screenshots, request ID if available, and the steps that caused the issue.</p>
    </div>
    <div class="utopia-troubleshooting-detail">
      <strong>Where to report</strong>
      <p>Use the Support Center in System Settings, submit an Incident Report, or contact the system administrator.</p>
    </div>
  </section>
</div>
