The Role & Permission component defines how access is controlled within the system by assigning roles and limiting what each user can view or perform.

It ensures that users only have access to the features and data relevant to their responsibilities.  

Overview

Administrators manage access by:

  • Assigning roles to users
  • Controlling visibility of system modules
  • Restricting access to sensitive data
  • Ensuring proper authorisation across the system

Role Definition

A role represents a user's responsibility within the organisation.

Each role determines which modules and features are accessible to the user.

Common Roles

Admin

Administrators have complete control over the system, including access to all modules, user data, and configuration settings.

Manager

Managers have access to approval functions, reporting tools, and team monitoring features.

Director

Directors have access to high-level approval and monitoring functions. They review and approve submissions escalated from managers and oversee cross-department activities.

Supervisor

Supervisors have access to operational modules and assist in reviewing and processing requests.

HR

HR staffs have access to employee-related modules, including attendance, leave, and employee records.

Accounts / Finance

Accounts have access to financial modules, including payment processing, claims verification, and reporting.

Credit

Credit space provides access to rental tracking and collection monitoring functions to oversee payment progress and account status.

OE (Operations Efficiency)

OE staff have access to operations attendance tracking and workforce efficiency monitoring tools.

Indoor Sales Operation

Indoor Sales staff have access to sales-related, tools, including document management, quotation tracking, and calculation tools.

Developer

Developers have access to system monitoring tools such as database usage and server performance.

Permission Control

Permissions define the actions that a role can perform within the system.

Examples include:

  • View information
  • Create records
  • Edit or update data
  • Approve or reject requests
  • Process transactions

Access Control

Access is managed based on roles to ensure that:

  • Users only see relevant modules
  • Sensitive information is protected
  • System operations remain secure and organised

Best Practices

  • Assign roles based on job responsibilities
  • Avoid granting full administrative access unnecessarily
  • Regularly review and update user roles
  • Ensure proper separation of duties
